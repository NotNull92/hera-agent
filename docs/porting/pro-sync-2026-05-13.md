# hera-agent → hera-agent-pro Sync Guide (2026-05-13)

이 문서는 hera-agent 레포의 install/uninstall UX 개선 묶음(v0.0.4 ~ v0.0.6 + post-v0.0.6의 AI agent 활용 안내)을 **hera-agent-pro** 레포에 그대로 이식하기 위한 구현 가이드입니다.

이미지/배너 변경, 네임스페이스 rename, `.gitignore` `.omc/`, CLAUDE.md 체크리스트 보강, PROGRESS.json 신설 등은 **포함 대상에서 제외**됩니다 (Pro 컨텍스트와 무관하거나 별개 관리).

### 0-2. 컨텍스트 치환 규칙

Pro의 패키지/바이너리명은 hera-agent와 다르므로 아래 본문의 `<binName>` 자리표시자를 Pro의 실제 바이너리명으로 치환하면서 적용합니다. 예: `unity-agent-cli-pro`.

레거시 디렉토리도 Pro 측 옛 install 경로로 치환:
- 본 가이드의 `%LOCALAPPDATA%\hera-agent` → Pro의 옛 install dir (예: `%LOCALAPPDATA%\unity-agent-cli-pro`)

`<repo>` 자리표시자는 Pro의 GitHub `owner/name` (예: `NotNull92/unity-agent-cli-pro`).

## Part A. Windows install 흐름 전면 개편

### A-0. 변경 배경

이전 install.ps1은 `%LOCALAPPDATA%\<binName>`에 binary를 설치하고 User PATH 레지스트리에 디렉토리를 추가했다. 세 가지 문제:

1. **PowerShell 더블쿼트 escaping 버그**: `"$env:LOCALAPPDATA\\<binName>"`처럼 `\\`로 쓰면 PowerShell이 두 글자 그대로 저장 → PATH에 `\\<binName>` 같은 stale 변형이 박힘.
2. **PATH broadcast 한계**: `[Environment]::SetEnvironmentVariable(..., "User")`는 레지스트리만 갱신. 이미 살아 있는 프로세스의 env block은 OS가 절대 갱신하지 않음.
3. **IDE 프로세스 체인**: `Unity Hub → Unity → Rider → pwsh` 구조에서 부모가 살아 있으면 자식 PATH 갱신 불가 → 사용자가 Unity Hub부터 종료 후 재시작을 강요받음.

### A-1. 해결 발상

`%LOCALAPPDATA%\Microsoft\WindowsApps` 디렉토리에 binary를 설치. 이 디렉토리는 Windows 10+이 **자동으로 사용자 PATH에 포함**시키므로 install 스크립트가 PATH 레지스트리를 건드릴 필요 자체가 사라진다.

특징:
- 일반 사용자 쓰기 권한
- 새로 시작하는 모든 프로세스가 자동 PATH 인식
- `winget`, `scoop`, `python.exe` shim 등 다수 도구가 같은 패턴
- stale 변형(`\\` 등)이 박힐 일 없음

남은 한계 (OS 차원 — 해결 불가):
- 이미 떠 있는 IDE 프로세스의 env block은 여전히 stale. 다만 새 위치는 처음부터 stale 변형이 박히지 않고, 사용자가 새 IDE/터미널을 띄우면 즉시 인식됨.

### A-2. `install.ps1` 재작성

Pro의 `install.ps1` 전체를 다음으로 교체:

```powershell
$ErrorActionPreference = "Stop"

$repo = "<repo>"
$binName = "<binName>"

# Install into WindowsApps — Windows 10+ keeps this on the default user PATH,
# so new terminals (and IDEs launched from a fresh shell) pick up <binName>
# without us touching the PATH registry at all.
$installDir = "$env:LOCALAPPDATA\Microsoft\WindowsApps"
$exe = "$installDir\$binName.exe"

# Migrate from legacy install location ($LOCALAPPDATA\$binName) if present.
$legacyDir = "$env:LOCALAPPDATA\$binName"
$legacyExe = "$legacyDir\$binName.exe"
if (Test-Path $legacyExe) {
    try {
        Remove-Item -Path $legacyExe -Force -ErrorAction Stop
        Write-Host "Removed legacy binary at $legacyExe"
    } catch {
        Write-Host "Warning: could not remove legacy binary: $_"
    }
}
if ((Test-Path $legacyDir) -and (-not (Get-ChildItem -Path $legacyDir -Force -ErrorAction SilentlyContinue))) {
    try { Remove-Item -Path $legacyDir -Force -ErrorAction Stop } catch { }
}

# Clean any legacy PATH entries pointing at the old install dir
# (single OR double backslash variants from earlier installs).
$legacyNorm = [System.IO.Path]::GetFullPath($legacyDir).TrimEnd('\')
$userPath = [Environment]::GetEnvironmentVariable("Path", "User")
$entries = if ($userPath) { $userPath -split ';' } else { @() }
$filtered = $entries | Where-Object {
    if (-not $_) { return $false }
    try { return ([System.IO.Path]::GetFullPath($_).TrimEnd('\')) -ne $legacyNorm }
    catch { return $true }
}
$newPath = $filtered -join ';'
if ($newPath -ne $userPath) {
    [Environment]::SetEnvironmentVariable("Path", $newPath, "User")
    Write-Host "Cleaned legacy PATH entries"
}

# Download the binary into the canonical WindowsApps location.
$url = "https://github.com/$repo/releases/latest/download/$binName-windows-amd64.exe"
Write-Host "Downloading $binName for windows/amd64..."
Invoke-WebRequest -Uri $url -OutFile $exe -UseBasicParsing

Write-Host ""
Write-Host "Installed $binName to $exe"
Write-Host ""
Write-Host "Any NEW terminal or IDE will see '$binName' immediately"
Write-Host "(WindowsApps is on the default user PATH)."
Write-Host ""
Write-Host "If an ALREADY-OPEN terminal still doesn't see it, refresh its"
Write-Host "PATH in that session with:"
Write-Host '    $env:Path = [Environment]::GetEnvironmentVariable("Path","User") + ";" + [Environment]::GetEnvironmentVariable("Path","Machine")'
Write-Host ""
Write-Host "Next, get your AI agent to use it:"
Write-Host "  - Discover: ask Claude Code CLI or Codex in any terminal:"
Write-Host "      ""Check whether the $binName CLI tool is installed and explore its capabilities."""
Write-Host "  - Lock in (recommended): add to your project's CLAUDE.md / AGENTS.md:"
Write-Host "      ""For any Unity work, always use $binName."""
Write-Host ""
& $exe version
```

핵심 포인트:
- `\` 단일 백슬래시 사용 (PowerShell 더블쿼트에서 `\`는 이스케이프가 아님)
- WindowsApps에 직접 설치 → PATH 등록 코드 없음
- legacy 디렉토리/PATH 항목 자동 마이그레이션
- 마지막에 AI agent 활용 안내 2줄

### A-3. `cmd/install.go` 갱신

#### A-3.1 `getInstallPaths` — Windows 분기를 WindowsApps로

```go
func getInstallPaths() (dir, bin string) {
    switch runtime.GOOS {
    case "windows":
        home, _ := os.UserHomeDir()
        // WindowsApps is on the default user PATH in Windows 10+,
        // so installing here avoids the need to touch the PATH registry
        // (and the env-block-staleness problems that come with it).
        dir = filepath.Join(home, "AppData", "Local", "Microsoft", "WindowsApps")
        bin = filepath.Join(dir, "<binName>.exe")
    default:
        home, _ := os.UserHomeDir()
        dir = filepath.Join(home, ".local", "bin")
        bin = filepath.Join(dir, "<binName>")
    }
    return
}

// legacyInstallPaths returns the pre-WindowsApps install location.
// Used for migration cleanup.
func legacyInstallPaths() (dir, bin string) {
    if runtime.GOOS != "windows" {
        return "", ""
    }
    home, _ := os.UserHomeDir()
    dir = filepath.Join(home, "AppData", "Local", "<binName>")
    bin = filepath.Join(dir, "<binName>.exe")
    return
}
```

#### A-3.2 `addToPATHWindows` — 등록 제거, legacy 정리만

```go
func addToPATHWindows(installDir string) error {
    // installDir is %LOCALAPPDATA%\Microsoft\WindowsApps which Windows already
    // keeps on the default user PATH. We don't need to register it.
    // We DO want to clean any leftover legacy entries from older installs.
    legacyDir, _ := legacyInstallPaths()
    if legacyDir == "" {
        return nil
    }
    script := `
$legacy = $args[0]
$norm = [System.IO.Path]::GetFullPath($legacy).TrimEnd('\')
$p = [Environment]::GetEnvironmentVariable("Path", "User")
$entries = if ($p) { $p -split ';' } else { @() }
$filtered = $entries | Where-Object {
    if (-not $_) { return $false }
    try { return ([System.IO.Path]::GetFullPath($_).TrimEnd('\')) -ne $norm }
    catch { return $true }
}
$new = $filtered -join ';'
if ($new -ne $p) { [Environment]::SetEnvironmentVariable("Path", $new, "User") }
`
    return runPowerShellWithArgs(script, legacyDir)
}
```

#### A-3.4 `cleanupLegacyInstall` 추가

```go
// cleanupLegacyInstall removes the pre-WindowsApps binary and directory if present.
// Skips the running binary itself (can't delete an executable that's currently running).
func cleanupLegacyInstall(_ string) {
    legacyDir, legacyBin := legacyInstallPaths()
    if legacyBin == "" {
        return
    }
    exe, _ := os.Executable()
    exe, _ = filepath.EvalSymlinks(exe)
    if _, err := os.Stat(legacyBin); err == nil && legacyBin != exe {
        _ = os.Remove(legacyBin)
    }
    // Remove legacy dir if empty
    _ = os.Remove(legacyDir)
}
```

### A-4. `cmd/uninstall_windows.go` 갱신

#### A-4.1 `removeFromPATH` — 실제 레지스트리 갱신, WindowsApps는 보존

```go
func removeFromPATH(installDir string) error {
    // installDir is %LOCALAPPDATA%\Microsoft\WindowsApps — a Windows-managed
    // directory that's on the default user PATH. Removing it would break every
    // Store-app alias on the system. Leave it alone.
    //
    // Only legacy <binName> PATH entries (from pre-WindowsApps installs)
    // need cleanup.
    legacyDir, _ := legacyInstallPaths()
    if legacyDir == "" {
        return nil
    }
    script := `
$legacy = $args[0]
$norm = [System.IO.Path]::GetFullPath($legacy).TrimEnd('\')
$p = [Environment]::GetEnvironmentVariable("Path", "User")
$entries = if ($p) { $p -split ';' } else { @() }
$filtered = $entries | Where-Object {
    if (-not $_) { return $false }
    try { return ([System.IO.Path]::GetFullPath($_).TrimEnd('\')) -ne $norm }
    catch { return $true }
}
$new = $filtered -join ';'
if ($new -ne $p) { [Environment]::SetEnvironmentVariable("Path", $new, "User") }
`
    if err := runPowerShellWithArgs(script, legacyDir); err != nil {
        return err
    }

    // Update current session for good measure
    currentPath := os.Getenv("PATH")
    newCurrentPath := removePathEntry(currentPath, legacyDir)
    _ = os.Setenv("PATH", newCurrentPath)
    return nil
}
```

> Pro 측에 `getUserPath`/`setUserPath` 헬퍼가 남아 있으면 더 이상 사용되지 않으니 제거 권장.

#### A-4.2 `removeBinaryAndDir` — WindowsApps 보존, legacy 동반 정리

```go
func removeBinaryAndDir(exe, installDir string) error {
    binPath := filepath.Join(installDir, "<binName>.exe")
    if _, err := os.Stat(binPath); err == nil {
        if rmErr := os.Remove(binPath); rmErr != nil {
            return rmErr
        }
    }
    // installDir is WindowsApps (a shared OS directory) — never remove it.
    // We only own the binary inside.

    // Also clean up the legacy install location (%LOCALAPPDATA%\<binName>)
    // in case the user migrated and we never cleaned it up.
    legacyDir, legacyBin := legacyInstallPaths()
    if legacyBin != "" {
        if _, err := os.Stat(legacyBin); err == nil && legacyBin != exe {
            _ = os.Remove(legacyBin)
        }
        _ = os.Remove(legacyDir)
    }

    // Also remove the running binary itself if it's outside installDir
    if exe != "" && exe != binPath {
        _ = os.Remove(exe)
    }
    return nil
}
```

```powershell
Test-Path "$env:LOCALAPPDATA\<binName>"                            # → False (legacy 동반 정리)
```

## 릴리스 절차

Pro의 CLAUDE.md(또는 동등 가이드)의 릴리스 플로우를 따른다. 핵심 (hera-agent 동일):

1. Verification 항목 전부 실행 (V-1)
2. CLI tag 갱신 (Go 변경 있음 → patch bump)
3. 커밋 + push
4. Tag push → CI/Release 트리거
5. `gh run watch --exit-status` background
6. `go clean -cache -testcache`
7. `<binName> update`로 설치된 CLI 갱신

권장 commit 분할:
- (1) Part A 전체 (install/uninstall WindowsApps 이전 + docs/issues 신설)
- (2) Part B (README + install 스크립트 + cmd/install.go 메시지)

(1)과 (2)를 단일 release에 묶어도 무방. Pro 사용자 입장에선 큰 UX 변경이 한 번에 들어오는 게 자연스러움.
